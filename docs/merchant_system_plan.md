# 流浪商人系统计划

## 背景

当前已有补给箱、精英宝箱、守点事件和升级三选一，但还没有能让玩家主动规划路线和构筑的交易事件。商人系统的目标是补上一个轻量经济循环和短期地图目标：玩家看到商人后，需要判断是否冒险靠近、由谁交易、队友是否保护。

多人游戏中不能用全局暂停商店。一个玩家买东西时让全队暂停，会打断其他玩家正在处理的战斗、救援、精英和走位，体验很差。

## 第一版定位

第一版做成“流浪商人事件 + 个人金币钱包”，不是完整局外经济系统。

- 商人是限时地图事件。
- 商人出现后持续 30 到 45 秒。
- 每 3 到 4 分钟尝试出现一次。
- 同一时间最多存在 1 个商人。
- 玩家靠近商人交易圈后，可以打开自己的购买面板。
- 战斗不暂停，其他玩家完全不受影响。
- 商品是个人商品，不做共享库存。
- 交易后商人不立刻消失，其他玩家仍可交易。
- 金币只在本局内有效，第一版不做局外货币。
- 金币获取全队同额增加，购买时只扣购买者自己的金币。

## 多人交互规则

核心规则：实时交易，不全局暂停。

- 靠近商人的玩家才能打开商店。
- 打开商店时世界继续运行。
- 玩家仍然可以移动，自动武器继续攻击。
- 商店 UI 使用小面板，支持 `1/2/3` 快速购买，`Esc` 关闭。
- 玩家离开交易圈后，商店自动关闭。
- 玩家倒地后，商店自动关闭，并且不能购买。
- 商人到期离开后，所有打开的商店自动关闭。

商品库存建议按玩家独立生成：

- 每个玩家看到自己的 3 个商品。
- 一个玩家购买不会影响其他玩家的商品。
- 房主/服务器负责生成和验证商品，客户端只显示和发送购买请求。

## 经济系统设计

核心规则：个人钱包，全队共享获取，个人消费。

- 每个玩家都有自己的 `gold`。
- 金币是本局资源，死亡结算后不带到局外。
- 任意玩家捡到金币时，所有当前参战玩家都获得同样金币。
- 商人购买只扣购买者自己的金币。
- 个人商品只影响购买者。
- 第一版不做团队公共钱包，避免一个玩家消费影响全队。

示例：

```text
地上金币值 8。
A 捡到后，A/B/C 每人 +8 金币。
A 在商人处花 45 买武器调校，只扣 A 的 45 金币，只强化 A 的武器。
```

这样能避免多人抢金币，也能避免共享钱包带来的误消费问题。

### 金币来源

第一版金币主要来自战斗和地图目标。

| 来源 | 金币规则 |
| --- | --- |
| 普通僵尸 | 8% 概率掉落 2 金币 |
| 冲刺者 | 12% 概率掉落 3 金币 |
| 爆炸者 | 14% 概率掉落 4 金币 |
| 精英怪死亡 | 全队直接 +25 到 +40 金币 |
| 宝箱打开 | 全队直接 +20 金币 |
| 守点完成 | 全队直接 +35 金币 |

普通怪不要每只都掉金币，避免后期地面物体太多。精英、宝箱、守点可以直接给全队金币，不必生成大量金币物体。

### 金币节奏目标

目标不是让玩家每次都买空，而是让商人出现时有选择压力。

- 3 分钟时：每名玩家大约 40 到 70 金币。
- 6 分钟时：每名玩家大约 100 到 160 金币。
- 10 分钟时：每名玩家大约 220 到 330 金币。

第一位商人出现时，玩家通常能买 1 件；第二位商人出现时，玩家通常能买 1 到 2 件。精英和守点奖励应该明显提升购买力。

## 受击打断规则

交易时被攻击不暂停、不无敌，直接打断交易。

- 玩家受到伤害时，商店面板立即关闭。
- 玩家正常掉血。
- 受击后 1.5 秒内不能重新打开商店。
- 商人不消失，玩家清怪后可以再回来交易。
- 如果玩家倒地，不能继续交易。
- 已经按下购买的交易立即结算，不回滚。
- 购买结算时必须再次验证金币、距离、倒地状态和受击冷却。

这个规则让商人变成有风险的目标，而不是免费安全点。

## 第一版商品

第一版只做 3 类个人商品，全部使用个人金币支付。

| 商品 | 价格 | 效果 | 设计目的 |
| --- | --- | --- | --- |
| 急救包 | 25 金币 | 回复自己 35% 最大生命 | 救急，降低死亡挫败 |
| 武器调校 | 45 金币 | 随机强化自己一件已有且未满级武器 | 主动推进武器路线 |
| 被动训练 | 50 金币 | 获得一个普通或优秀被动 | 补构筑短板 |

商品购买规则：

- 每个玩家每个商品每次商人只能买 1 次。
- 商品列表按玩家独立生成，一个玩家购买不会影响其他玩家。
- 急救包可以在满血时显示为不可购买，避免浪费。
- 武器调校在没有可升级武器时不可购买。
- 被动训练在可选被动池为空时不可购买。

如果当前条件不满足，商品可以显示为不可购买：

- 金币不足，商品不可购买。
- 当前生命已满，急救包不可购买。
- 没有可升级武器，武器调校不可购买。
- 被动池为空，被动训练不可购买。

### 第二版商品

第一版稳定后，再考虑更强商品：

| 商品 | 价格 | 效果 | 注意事项 |
| --- | --- | --- | --- |
| 遗物盒 | 90 金币 | 获得随机遗物 | 当前遗物是团队共享，需标清“全队获得” |
| 进化线索 | 120 金币 | 如果满足条件，提供武器进化选项 | 需要复用进化候选过滤 |
| 团队补给 | 140 金币 | 全队回血或清屏 | 属于团队商品，暂不进第一版 |

因为当前遗物系统是团队共享，遗物商品第一版先不做，避免“个人购买却全队获益”的价格和解释问题。

## 关键防错系统

这些不是额外内容，而是第一版必须一起做的稳定性规则。否则商店很容易被刷、重复扣款、多人不同步或造成 UI 冲突。

### 商品锁定

每个商人必须有唯一 `merchant_id`。

- 商人生成时分配 `merchant_id` 和 `offer_seed`。
- 每个玩家第一次打开该商人时，房主生成该玩家的 3 个商品。
- 生成后的商品列表按 `merchant_id + peer_id` 锁定。
- 玩家关闭商店、离开交易圈、重新进入、受击打断后，再打开仍看到同一组商品。
- 断线重连后，如果商人还存在，仍恢复同一组商品和已购买状态。
- 商人离开后，相关商品列表和购买状态清理。

这样可以防止玩家通过反复进出商人圈、关闭重开或断线重连来刷新商品。

### 购买请求幂等

多人网络下，同一个购买请求可能因为延迟或重发被房主收到多次。购买必须幂等。

购买请求至少包含：

- `merchant_id`
- `offer_id`
- `offer_index`
- `peer_id`

房主处理购买时：

- 先检查 `merchant_id` 是否仍有效。
- 再检查该 `peer_id` 的该 `offer_id` 是否已经购买。
- 如果已经购买，直接忽略重复请求，不扣钱、不重复发奖励。
- 如果未购买且验证通过，先标记已购买，再扣金币和发奖励。

建议用 `purchased_offer_ids_by_peer[peer_id][offer_id] = true` 记录。

### 商品合法性检查

商品生成和购买结算都要各检查一次。

生成时检查：

- 没有可升级武器时，不生成武器调校。
- 满血时可以不生成急救包，或生成但标记不可购买。
- 被动池为空时，不生成被动训练。
- 如果三类商品都不可用，使用 fallback 商品。

购买时再次检查：

- 金币是否足够。
- 玩家是否还在交易圈内。
- 玩家是否倒地或等待下一局。
- 玩家是否处于受击重开冷却。
- 商品是否仍然适用于当前状态。

推荐 fallback：

- `弹药保养`：25 金币，本局短时间内伤害 +10%，持续 30 秒。
- 如果暂不做临时状态，fallback 用 `急救包`，但满血时不可购买。

### 金币通胀控制

金币必须有产出预算，不能只按怪物数量线性增长。

第一版建议：

- 同屏金币物体软上限：80。
- 全局金币物体硬上限：120。
- 普通怪金币掉落受每分钟预算限制。
- 精英、宝箱、守点是主要金币来源。
- 普通怪金币掉落只负责补足小额收入，不承担主要经济。

建议初始预算：

| 时间段 | 普通怪金币预算 |
| --- | --- |
| 0 到 3 分钟 | 每分钟最多 24 金币 |
| 3 到 6 分钟 | 每分钟最多 36 金币 |
| 6 到 10 分钟 | 每分钟最多 48 金币 |
| 10 分钟后 | 每分钟最多 60 金币 |

预算指“全队同额获得的金币值”，不是每个玩家额外翻倍计算。

### 失败反馈

购买失败必须给明确原因，不能只是不响应。

需要支持的失败原因：

- 金币不足。
- 离商人太远。
- 刚被攻击，交易冷却中。
- 已购买。
- 倒地不能购买。
- 商品当前不可用。
- 商人已经离开。

UI 显示短提示即可，例如 `金币不足`、`交易被打断`、`离商人太远`。

### UI 冲突规则

商人 UI 不能和升级三选一、宝箱奖励、结算界面互相抢输入。

规则：

- `choosing_upgrade == true` 时不能打开商人。
- 商人面板打开时，如果触发升级三选一，商人面板自动关闭，升级 UI 优先。
- 游戏结算时关闭所有商人面板。
- 玩家倒地时关闭商人面板。
- 玩家打开商店时仍可移动和自动攻击，但 `1/2/3` 优先用于商店购买。
- 如果升级 UI 打开，`1/2/3` 优先用于升级选择。

### 中途加入和断线规则

第一版规则：

- 中途加入玩家金币为 0。
- 中途加入玩家可以看到当前商人，但没有历史商品。
- 玩家断线后，房主保留该玩家本局金币和商人购买状态一段时间。
- 同一 peer 重连后恢复金币、商品锁定和已购买状态。
- 如果无法可靠识别重连玩家，按新玩家处理，金币为 0。

### 商人生成位置

商人不能刷得太安全，也不能刷在明显不可用的位置。

生成时要避开：

- 地图边界太近。
- 敌人密度极高的位置。
- 玩家完全够不到的位置。
- 与宝箱、补给、守点严重重叠的位置。

推荐规则：

- 距离队伍焦点 450 到 750 像素。
- 距离地图边界至少 120 像素。
- 距离现有事件至少 160 像素。
- 周围 180 像素内敌人数量超过阈值时重新找点。

## 执行阶段

### P0. 稳定可用闭环

目标：先跑通经济、商人、购买验证和多人不暂停。

任务：

1. 新增个人金币状态和 HUD 显示。
2. 新增金币掉落和全队同额获得。
3. 新增商人事件生成、绘制和交易圈。
4. 新增轻量商人 UI，不复用暂停升级 UI。
5. 实现 3 个第一版商品：急救包、武器调校、被动训练。
6. 实现受击打断和 1.5 秒重开冷却。
7. 实现房主权威购买验证。
8. 实现商品锁定和重复请求防护。

验收：

- 多人中一个玩家购物不影响其他玩家。
- 金币获取全队同额增加。
- 购买只扣购买者金币。
- 同一个商品不能重复购买。
- 反复打开商店不能刷新商品。
- 受击会关闭商店。

### P1. 平衡与可读性

目标：让商人值得去，但不能每次买空。

任务：

1. 增加金币产出预算和金币物体上限。
2. 调整普通怪、精英、宝箱、守点金币收益。
3. 增加购买失败原因提示。
4. 完善商人生成位置过滤。
5. 记录本局金币产出和商品购买率。

验收：

- 第一位商人通常能买 1 件。
- 第二位商人通常能买 1 到 2 件。
- 玩家看得懂为什么买不了。
- 后期金币物体不会堆满地图。

### P2. 内容扩展

目标：在稳定经济上增加更有肉鸽味的商品。

候选：

- 遗物盒。
- 进化线索。
- 团队补给。
- 折扣商人。
- 诅咒商品。

进入 P2 前提：

- P0/P1 验证稳定。
- 金币产出曲线可控。
- 多人同步没有重复购买或金币不同步问题。

## 暂不做

第一版明确不做这些内容：

- 局外金币。
- 共享金币钱包。
- 玩家之间交易金币。
- 折扣。
- 刷新商品。
- 复杂库存。
- 共享商品抢购。
- 复杂团队商品。
- 全员暂停。
- 投票暂停。
- 交易时无敌。
- 常驻基地商店。

这些内容等事件型商人验证好玩后再扩展。

## 实现方案

### 新增状态对象

新增 `scripts/entities/gold_orb_state.gd`。

建议字段：

- `position`
- `value`
- `lifetime`

普通怪掉落少量金币时生成金币物体。精英、宝箱、守点奖励可以直接给全队金币，不生成物体。

新增 `scripts/entities/merchant_event_state.gd`。

建议字段：

- `merchant_id`
- `position`
- `radius`
- `lifetime`
- `offer_seed`

商人本身只存位置、范围、剩余时间和随机种子，不直接处理购买逻辑。

### 玩家金币状态

需要为每个玩家维护个人金币。

第一版使用独立经济模块维护，不把钱包规则直接塞进主流程：

- `scripts/economy/gold_wallets.gd`
- `scripts/economy/gold_drop_policy.gd`

`GoldWallets` 负责每名玩家的局内金币余额、发放、扣费、序列化和快照恢复。
`GoldDropPolicy` 负责普通怪掉落概率、每分钟预算和金币物体上限。
`game.gd` 只在击杀、拾取、宝箱和守点完成时调用这些模块，并把结果纳入世界快照。

重开一局时：

- 所有参战玩家金币清零。
- 中途加入玩家从当前全队平均值或 0 开始，第一版建议 0，避免后加入刷资源。

金币增加函数：

- `_grant_gold_to_active_players(amount: int)`：所有当前参战且非等待玩家获得同样金币。
- `_grant_gold_to_peer(peer_id: int, amount: int)`：只给单个玩家，用于后续特殊奖励。

商店购买只调用扣购买者金币：

- `_try_spend_gold(peer_id: int, amount: int) -> bool`

### 扩展地图事件导演

扩展 `scripts/events/map_event_director.gd`：

- 增加商人计时器。
- 增加 `FIRST_MERCHANT_TIME`，建议 150 秒。
- 增加商人间隔，建议 180 到 240 秒。
- 增加 `MAX_ACTIVE_MERCHANTS = 1`。
- `update()` 返回 `merchant: true/false`。

### 主流程接入

扩展 `scripts/game/game.gd`：

- 新增 `gold_orbs` 数组。
- 新增 `gold_wallets` 经济钱包对象。
- 新增 `gold_drop_policy` 金币掉落策略对象。
- 新增 `merchant_events` 数组。
- 新增 `merchant_shop` 商店状态对象，锁定商品、记录购买和受击重开冷却。
- 新增 `merchant_panel_view` 商人面板视图对象，负责按钮和显示，不把 UI 细节堆在主流程。
- 击杀普通怪时按敌人类型概率生成金币。
- 精英死亡、宝箱打开、守点完成时直接给全队金币。
- `_update_xp_orbs()` 类似地新增 `_update_gold_orbs(delta)`。
- `_update_map_events()` 中生成和更新商人。
- `_draw()` 中绘制商人和交易圈。
- `_draw()` 中绘制金币物体。
- 新增 `_update_merchant_events(delta)`。
- 新增 `_open_local_merchant_panel(merchant)`。
- 新增 `_close_local_merchant_panel()`。
- 新增 `_apply_merchant_purchase(peer_id, merchant_id, offer_id, offer_index)`。
- HUD 显示本地玩家金币。

商店 UI 不应复用会暂停游戏的升级三选一流程。建议新增轻量商人面板，不设置 `choosing_upgrade = true`。

### 受击打断接入

需要在玩家实际受到伤害时通知商人系统。

推荐方式：

- 在主循环里调用 `take_damage()` 前记录玩家血量。
- 如果调用后血量下降，则执行 `_interrupt_merchant_for_peer(peer_id)`。
- `_interrupt_merchant_for_peer()` 关闭该玩家商店，并设置 `merchant_reopen_cooldowns[peer_id] = 1.5`。

这样不用先改 `Player` 成信号式结构，改动更集中。

### 网络同步

多人时由房主/服务器权威处理：

- 商人生成由房主决定。
- 金币掉落、金币拾取和金币奖励由房主决定。
- 金币拾取后由房主给所有当前参战玩家增加同样金币。
- 商人位置、剩余时间通过世界快照同步。
- 每个玩家的金币余额通过世界快照同步。
- 每个玩家的商品列表由房主生成并同步给对应玩家。
- 客户端发送购买请求，包含 `merchant_id`、`offer_id` 和商品索引。
- 房主验证玩家状态、距离、冷却、商品是否可买、金币是否足够、是否已经购买过。
- 验证通过后由房主扣金币并应用奖励。

需要避免客户端自己直接结算购买，否则容易出现不同步。

## UI 建议

商人面板要比升级三选一更轻：

- 面板靠屏幕右侧或下方，不遮挡中心战斗。
- 显示商人剩余时间。
- 显示本地玩家金币余额。
- 显示 3 个商品、价格、效果。
- 支持 `1/2/3` 购买。
- 支持 `Esc` 关闭。
- 被攻击时显示短提示：`交易被打断`。
- 交易圈内显示明显边框和倒计时。

## 验收标准

- 多人中一个玩家打开商店时，其他玩家不暂停、不弹窗。
- 打开商店的玩家仍能移动，战斗继续运行。
- 交易玩家受伤后商店立刻关闭，并进入短暂重开冷却。
- 商品购买只影响购买者。
- 一个玩家购买不会移除其他玩家的商品。
- 任意玩家捡到金币时，所有当前参战玩家都获得同样金币。
- 商人购买只扣购买者自己的金币。
- 商人到期后自动离开。
- 客户端不能在距离外、倒地、冷却中、金币不足或商品已购买时成功购买。
- 第一版不依赖局外金币，也能形成清楚的局内经济循环。
